Manual Testing Course Content

Manual testing is a crucial aspect of software quality assurance, ensuring that software applications meet their intended functionality and are free of defects before reaching users. In this course, you will explore the key concepts, techniques, and best practices in manual testing through a structured curriculum that includes lectures, hands-on exercises, real-world examples, and interactive discussions.

  • What is software testing?
  • Importance of software testing in the SDLC (Software Development Life Cycle)
  • Types of testing: manual vs. automated
  • Common testing terminology and concepts

  • Testing principles and fundamentals
  • Testing life cycle
  • Testing levels (unit testing, integration testing, system
  • testing, user acceptance testing)
  • Testing types (functional, non-functional, etc.)
  • Test Planning and Test Strategy
  • Test planning process
  • Creating a test strategy
  • Test plan documentation
  • Test environment setup

  • Test case development process
  • Test case design techniques (equivalence partitioning, boundary value analysis, decision tables, etc.)
  • Writing effective and clear test cases
  • Traceability between requirements and test cases
  • Test Execution and Defect Management
  • Test execution process
  • Executing test cases manually
  • Defect identification and reporting
  • Defect life cycle
  • Regression testing

  • Understanding functional requirements
  • Black-box testing techniques
  • Positive and negative testing
  • Smoke testing and sanity testing
